# Step 1 - 개발 업무 분석

## 확인된 전제
- 배포 대상: `https://kimdasomee.github.io`
- 저장소: `https://github.com/kimdasomee/kimdasomee.github.io.git`
- 구현 범위: 정적 `HTML`, `CSS`, `JavaScript`만 사용
- 추가 기능: 상단 `Games` 메뉴와 키보드, 모바일 터치로 조작하는 지렁이 게임
- 현재 작업공간 기준: 저장소 소스는 아직 보이지 않고, 현재 브랜치는 초기 상태로 보임
- 프로필 참고 자료: `https://ujutokki.github.io/resume/`

## 반영 방향
- 개인 프로페셔널 웹사이트는 반응형 단일 페이지 또는 최소 페이지 구조로 설계
- 상단 내비게이션에 `Games` 메뉴를 별도로 두어 게임 영역으로 이동 가능하게 구성
- 게임은 본문 흐름을 해치지 않도록 독립 섹션 또는 모달/패널 형태로 배치
- 프로필 참고 자료의 구체 내용은 이 환경에서 확인되지 않았으므로, 내용 항목은 우선 템플릿으로 두고 `[사람 확인 필요]` 표시

## 가장 안전한 첫 루프 추천
**1번 루프**를 가장 안전한 첫 루프로 추천한다.

이유:
- 아직 저장소가 초기 상태라서, 기능 구현보다 먼저 뼈대를 세우는 편이 리스크가 가장 낮다.
- 프로필 내용이 일부 불확실하므로 `[사람 확인 필요]`를 두고도 진행 가능하다.
- `Games` 메뉴와 게임 섹션의 위치를 먼저 확정하면 이후 게임 추가 작업이 흔들리지 않는다.
- 반응형 구조가 먼저 잡히면 나중에 콘텐츠와 게임을 넣어도 전체 레이아웃 수정이 적다.

## 확인 필요 사항
- 프로필 참고 자료의 실제 항목 구성과 문구: `[사람 확인 필요]`
- 게임의 세부 룰: 점수 방식, 종료 조건, 난이도 상승 여부 `[사람 확인 필요]`
- Games 메뉴의 노출 방식: 상단 앵커 이동인지, 별도 섹션인지, 카드형 진입인지 `[사람 확인 필요]`
- 연락처/CTA에 넣을 실제 링크와 문구 `[사람 확인 필요]`
